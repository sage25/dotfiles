## 1. 基础操作

### 通用

| 按键 | 作用 |
|------|------|
| \<Space\> | Leader 键（几乎所有自定义快捷键的前缀） |
| jk（插入模式） | 退出插入模式（等效 Esc） |

### 窗口导航

按 \<Space\> + 方向键在分屏窗口间跳转：

| 按键 | 作用 |
|------|------|
| \<Space\>h | 跳转到左边的窗口 |
| \<Space\>j | 跳转到下面的窗口 |
| \<Space\>k | 跳转到上面的窗口 |
| \<Space\>l | 跳转到右边的窗口 |

### 自动保存

每 5 分钟自动保存当前文件（仅在文件被修改时）。

### 自动恢复光标位置

打开文件时光标自动跳转到上次退出时的位置。

### 粘贴模式

| 按键 | 作用 |
|------|------|
| \<F6\> | 切换粘贴模式（粘贴代码时防止自动缩进打乱格式） |

## 2. 配色主题 — Catppuccin

猫猫配色，默认启用 Mocha 风味。

**无需手动操作**，启动即生效。支持 Treesitter、LSP、Telescope 等所有插件的配色。

---

## 3. 状态栏 — Lualine

屏幕底部的状态栏，从左到右显示：

| 区域 | 显示内容 |
|------|----------|
| 左侧 | 当前模式（NORMAL、INSERT、VISUAL 等）|
| 左中 | 当前文件名 |
| 中间 | LSP 诊断信息（错误数、警告数） |
| 右中 | 编码（UTF-8）和换行符（UNIX/DOS） |
| 右侧 | 文件类型 |
| 最右 | 光标位置百分比 |

非活动窗口只显示文件名和光标位置。

---

## 4. 缩进参考线 — Indent Blankline

在缩进位置显示垂直参考线，帮助看清代码层级。

**无需手动操作**，打开代码文件自动显示。

## 5. 按键提示 — Which-key

按下 \<Space\>（Leader 键）后等待半秒，屏幕下方会弹出可用的快捷键列表。

**无需记快捷键**——按 \<Space\> 后等一会就能看到能按什么。

## 6. 自动配对括号 — nvim-autopairs

输入 ( [ { ' " 时自动补全另一半。

**无需手动操作**，插入模式下自动生效。

## 7. 环绕编辑 — nvim-surround

快速添加、修改、删除配对标点符号（括号、引号、HTML 标签等）。

### 添加环绕 ys
ys{motion}{字符}

| 操作 | 结果 |
|------|------|
| ysiw" | 在光标所在单词外包上双引号：hello → "hello" |
| ysiw( | hello → (hello) |
| ysiw[ | hello → [hello] |
| yssb | 整行包上括号：hello → (hello) |

### 删除环绕 ds
ds{字符}

| 操作 | 结果 |
|------|------|
| ds" | "hello" → hello |
| ds( | (hello) → hello |

### 修改环绕 cs
cs{原字符}{新字符}

| 操作 | 结果 |
|------|------|
| cs"' | "hello" → 'hello' |
| cs([ | (hello) → [hello] |

### 可视模式

选中文本后按 S{字符} 给选中内容包上环绕。

## 8. 快速注释 — Comment.nvim

| 按键 | 作用 |
|------|------|
| gcc | 注释/取消注释当前行 |
| gc{motion} | 注释/取消注释光标经过的行 |
| gcb | 注释/取消注释代码块 |
| gb | 注释/取消注释选中区域（可视模式） |

示例：
- gcap — 注释整个段落
- gcj — 注释当前行和下一行
- 选中多行后按 gb — 块注释

## 9. 文件树 — Neo-tree

### 打开 / 关闭

| 方式 | 作用 |
|------|------|
| 启动时 nvim 文件名 | 自动打开文件树 |
| 启动时 nvim（无参数） | 显示启动页，不打开文件树 |
| :Neotree | 手动打开/关闭文件树 |

### 文件树内快捷键

| 按键 | 作用 |
|------|------|
| o / Enter | 打开文件/展开目录 |
| a | 新建文件 |
| d | 删除文件 |
| r | 重命名文件 |
| c | 复制文件 |
| m | 移动文件 |
| p | 粘贴 |
| s | 在垂直分屏中打开 |
| S | 在水平分屏中打开 |
| H | 显示/隐藏隐藏文件 |
| ? | 查看所有快捷键 |
| q | 关闭文件树 |

### 自动关闭

当文件树是最后一个窗口时，关闭其他窗口会自动关闭文件树。

## 11. 模糊查找 — Telescope

最常用的搜索工具。

### 快捷键

| 按键 | 作用 |
|------|------|
| \<Space\>ff | 搜索文件名（模糊匹配） |
| \<Space\>fg | 搜索文件内容（live_grep，实时全文搜索） |
| \<Space\>fb | 在已打开的缓冲区中切换 |

### 结果列表操作

| 按键 | 作用 |
|------|------|
| \<C-j\> | 向下移动选择 |
| \<C-k\> | 向上移动选择 |
| Enter | 打开选中项 |
| \<C-c\> / Esc | 关闭 Telescope |

### 扩展

- **ui-select**：Neovim 内置的 vim.ui.select 弹窗改用 Telescope 风格
- **frecency**：find_files 会根据打开频率和最近使用时间排序，常用文件排在前面

## 12. LSP（语言服务）— Mason + lspconfig

### 安装语言服务
:Mason

在 Mason 窗口中：
- 按 i 搜索要安装的语言服务
- 找到后按 I（大写）安装
- 按 X 卸载

当前已安装：
- **gopls**（Go 语言服务，通过 mason-lspconfig 的 ensure_installed 自动安装）

### gopls 自动启动

打开 .go 文件时自动启动 gopls，不需要手动操作。

### LSP 功能

| 按键 | 作用 |
|------|------|
| K | 显示光标下符号的文档（hover） |
| gd | 跳转到定义 |
| grr | 查看所有引用 |
| gra | 代码操作（重构、自动修复等） |
| grn | 重命名符号 |
| gO | 查看文档符号列表 |
| [d | 上一个诊断错误 |
| ]d | 下一个诊断错误 |

\> 以上为 Neovim 0.11+ 内置默认快捷键。

### 状态栏显示

状态栏中间会显示 LSP 诊断信息（错误数、警告数）。

## 13. 自动补全 — Blink.cmp

LSP 补全框架，提供代码补全、代码片段、路径补全。

### 补全行为

- 输入时自动弹出补全菜单
- **不自动选中**任何候选项
- 按 Tab 开始选中

### 快捷键

| 按键 | 作用 |
|------|------|
| \<Tab\> | 选中下一个候选项；无选中时选中第一个 |
| \<S-Tab\> | 选中上一个候选项 |
| \<CR\> | 确认选中的候选项 |

### 补全来源

| 来源 | 说明 |
|------|------|
| lsp | LSP 提供的代码补全（变量名、函数名等） |
| snippets | 代码片段（如 for 展开为 for 循环） |
| path | 文件路径补全（如 ./src/） |

## 14. 语法高亮 — Treesitter

基于语法树的精准高亮（比传统正则高亮更准确）。

**无需手动操作**，打开代码文件自动生效。

### 当前安装的语言解析器

C、C++、Go、Lua、Vim、Markdown

### 手动管理

| 命令 | 作用 |
|------|------|
| :TSInstall {语言} | 安装指定语言的解析器 |
| :TSUpdate | 更新所有已安装的解析器 |
| :TSInstallInfo | 查看可安装的解析器 |

## 15. Git 集成 — Fugitive + Gitsigns

### Gitsigns — 行级 Git 标记

在行号栏显示：

| 标记 | 含义 |
|------|------|
| │（绿色） | 新增的行 |
| │（红色） | 删除/修改的行 |
| ┃（蓝色） | 修改的行 |

内置快捷键（默认）：

| 按键 | 作用 |
|------|------|
| ]c | 跳转到下一个改动块 |
| [c | 跳转到上一个改动块 |
| \<leader\>hs | 暂存（stage）当前块 |
| \<leader\>hr | 撤销当前块的修改 |
| \<leader\>hp | 预览当前块的修改 |

### Fugitive — Git 命令

| 命令 | 作用 |
|------|------|
| :Git | 执行任意 Git 命令（如 :Git log、:Git diff） |
| :G | :Git 的简写 |
| :Gdiffsplit | 在分屏中显示当前文件的 diff |
| :Gblame | 逐行显示 blame 信息 |
| :Glog | 显示当前文件的提交历史 |
| :Gstatus | 查看 Git 状态（按 - 暂存/撤销，按 cc 提交） |

查看 git log 后，按 Enter 在分屏中查看该提交的改动。

## 16. LeetCode 刷题

### 打开 LeetCode
:Leet

### 主要功能

| 操作 | 描述 |
|------|------|
| 题目列表 | 浏览所有可用题目 |
| 搜索 | 按标题或编号搜索题目 |
| 选择题目 | 选中后自动创建代码文件 |
| 编写代码 | 使用 Go 语言答题 |
| :Leet submit | 提交当前题目的答案 |
| :Leet test | 运行测试 |
| :Leet reset | 重置代码到初始状态 |
| :Leet info | 查看题目信息 |

### 提示

LeetCode 中文站已启用（cn.enabled = true），会翻译题目描述。
